# Post-Release Checklist

After publishing:

```txt
- Confirm GitHub Release downloads artifact.
- Confirm Modrinth page shows correct version.
- Confirm CurseForge file enters review or is approved.
- Confirm Minecraft version and loader metadata are correct.
- Update README badges/links if needed.
- Create follow-up task for release feedback.
- Monitor crash reports/comments.
- Record any platform-specific publishing issues.
```

## First 24 Hours

```txt
- Watch for startup crashes.
- Watch for save/load data loss reports.
- Watch for broken dependencies.
- Watch for wrong loader/version metadata.
```

## Follow-Up

```txt
- Create hotfix branch if needed.
- Update known issues.
- Triage feedback.
- Plan next alpha/beta.
```
