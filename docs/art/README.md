# Curious Bees Art and Asset Pipeline

This folder defines how Curious Bees should handle art assets, placeholder assets, Blockbench models, and future AI-assisted asset workflows.

The goal is not to make polished art part of the early critical path.

The goal is to make sure that, when assets become necessary, they are created consistently, stored in the right places, and integrated without distracting from genetics, breeding, and gameplay implementation.

## Core Principle

Gameplay comes first.

Assets should support implementation, not block implementation.

Use placeholder assets until the related gameplay is stable.
